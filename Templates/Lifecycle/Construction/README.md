# Construction/
**Source:** Templates/Lifecycle/Construction/README.md

Tactical artifacts created during active development. This directory starts empty and grows organically.

## Subdirectories

### Test-Plans/
**Purpose:** Proposed test cases and strategies for specific features.
**Template:** `Test-Plan-{feature}.md`

### Design-Decisions/
**Purpose:** Technical decisions with context and rationale.
**Template:** `Decision-{NNNN}-{title}.md`

### Sprint-Retros/
**Purpose:** Retrospective summaries from completed sprints.
**Template:** `Retro-{sprint-name}.md`

### Tech-Debt/
**Purpose:** Identified technical debt for later resolution.
**Template:** `Debt-{title}.md`

## Usage Notes
- Files are created by the AI assistant when triggers are detected
- All files should be committed to version control
- This directory reflects the "how" of building
- See `Inception/` for strategic "what" decisions
